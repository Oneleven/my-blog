到底什么是函数执行环境

发表于 2018-04-11 | 分类于 JavaScript

####执行上下文（也称执行环境）

js中有3种不同的运行环境，为了表示不同的运行环境，js中有执行上下文（Execution context，EC）的概念。就是当js代码执行的时候，会进入不同的执行上下文，这些执行上下文就构成了一个执行上下文栈（Execution context stack，ECS）

  • Global Code：js开始运行的默认环境
  • Function Code：代码进入一个js函数
  • Eval Code：使用eval()执行代码

代码首先进入Global Execution Context全局执行上下文，然后依次进入outer,inner,foo的执行上下文。执行上下文栈如下图。

  • js代码执行的时候，第一个进入的总是Global Execution Context ，所以他总是在ECS的最底部。
  • 函数的每次调用都会创建一个新的执行上下文，函数的环境就会被推入Execution Context Stack顶部。在函数执行完后，栈将其环境弹出，把控制权返回给之前的执行环境。

####Execution Context

每个Execution Context都有3个重要的属性

  • 变量对象（Variable object，VO）：变量对象是与执行上下文相关的特殊的对象，存储了在上下文中定义的变量和函数声明。当js运行代码的时候如果要寻找变量，首选会查找VO。VO包含以下信息
    • 变量
    • 函数声明
    • 函数的形参
  • 作用域链（Scope chain）
  • this
活动对象（Activation object，AO）

在global全局上下文中，变量对象也是全局对象自身。在函数上下文中，VO是不能直接访问的，此时变量对象被表示为活动对象AO，活动对象是在进入函数上下文的时候被创建的，它通过函数的arguments属性初始化

Arguments Objects 是函数上下文里的活动对象AO中的内部对象，它包括下列属性：

  1. callee：指向当前函数的引用
  2. length： 真正传递的参数的个数
  3. properties-indexes：就是函数的参数值(按参数列表从左到右排列)

####Execution Context的内部过程

当js代码执行的时候，js解释器会创建Execution Context，这里有两个阶段

  • 函数被调用，但是开始执行函数内部代码之前
    • 创建Scope chain，作用域链
    • 创建VO/AO
      • 根据函数的参数，创建并初始化参数对象arguments object
      • 扫描内部代码，查找函数声明，并将函数名和函数引用存入VO/AO中
      • 扫描内部代码，查找变量声明，并把变量名存入VO/AO中，并初始化为undefined
    • 设置this值
  • 函数内部代码执行阶段
    • 设置变量的值，函数的引用，然后执行代码
